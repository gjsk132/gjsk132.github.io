# Agent Working Rules

이 파일은 이 프로젝트에서 Codex와 AI 코딩 도구가 반복해서 지켜야 하는 작업 규칙입니다.

## 명령 실행 규칙

- 사용자는 기본적으로 Git Bash를 사용합니다. 사용자에게 안내하는 명령어와 문서 예시는 Git Bash 기준으로 작성합니다.
- Windows PowerShell 전용 명령은 꼭 필요한 경우에만 사용하고, 사용했다면 PowerShell 기준이라고 명시합니다.
- 사용자가 직접 요청하기 전에는 `npm run dev`를 실행하지 않습니다.
- 사용자가 직접 요청하기 전에는 `git status`를 실행하지 않습니다.
- `git commit`과 `git push`는 절대 임의로 실행하지 않습니다.
- 빌드, 린트, 테스트가 필요해 보이면 먼저 어떤 검증이 필요한지 간단히 설명하고, 사용자가 요청하거나 승인한 범위에서 실행합니다.

## 인코딩 규칙

- 프로젝트 문서는 UTF-8로 작성합니다.
- 한글이 깨져 보이면 파일 내용 문제인지 터미널 출력 문제인지 먼저 구분합니다.
- 사용자에게 파일 확인 명령을 안내할 때는 Git Bash 기준으로 `cat`, `sed`, `rg`를 우선 사용합니다.
- PowerShell에서 한글 확인이 필요할 때만 `Get-Content -Encoding utf8`처럼 인코딩을 명시합니다.

## 작업 방식

- 프로젝트를 수정하기 전에 관련 파일을 먼저 읽고 기존 구조와 스타일을 따릅니다.
- 불확실한 기획 판단, 문구 방향, 정보 구조처럼 추가 설명이 필요한 부분은 혼자 결정하지 말고 사용자에게 질문합니다.
- 단순 구현 세부사항은 기존 코드 패턴에 맞춰 보수적으로 결정합니다.
- 사용자가 만든 변경사항을 되돌리지 않습니다.

## UI 확인 규칙

- HTML 페이지 전환, 필터, 디자인 변경, 보기 형태 전환이 들어간 작업은 변경 후 한 번씩 화면 상태를 확인할 수 있게 합니다.
- 특히 홈의 게시글형/타임라인형 전환, 필터 적용, 검색 패널 표시처럼 레이아웃이 바뀌는 기능은 버벅임이나 겹침이 없는지 확인 대상으로 봅니다.
- 사용자가 개발 서버 실행을 원하지 않는다고 명시했거나 아직 요청하지 않았다면, 확인 방법만 남기고 서버는 실행하지 않습니다.

## 금지 사항

- 사용자 요청 없이 commit/push 하지 않습니다.
- 사용자 요청 없이 개발 서버를 계속 켜 두지 않습니다.
- 토큰을 많이 쓰는 반복 상태 확인 명령을 습관적으로 실행하지 않습니다.

## 반복 요청 기록

- 불필요한 `npm run dev` 실행을 피합니다.
- 불필요한 `git status` 확인을 피합니다.
- `git commit`, `git push`는 하지 않습니다.
- Git Bash 기준으로 설명하고 명령어를 작성합니다.
- 한글 깨짐을 피하기 위해 UTF-8 기준으로 파일을 다룹니다.
- UI 변경 후에는 페이지 전환, 필터, 보기 형태 전환의 흔들림이나 겹침을 확인할 수 있게 합니다.
- 추가 설명이나 결정이 필요한 부분은 임의로 정하지 않고 사용자에게 묻습니다.
